

# 1. Статические файлы

Любое современное веб-приложение помимо html файлов также требует файлы css, js картинки. Эти файлы далее будем называть **статическими**.

Джанго предоставляет удобный механизм подключения этих файлов при локальной разработке. 

### 1. Cоздание статичного файла

1. Создайте папку static в корне проекта. Она может назваться по-другому. Главное, чтобы потом ее название было указано в настройках. Обычно используется название static.
2. Внутри папки static создайте папку css. Название также может быть другим. 
3. Внутри папки css создайте файл main.css (название может быть другим) и поместите туда стиль для примера. 
![фвыа](http://images.na4u.ru/static/static/1.png)


### 2. Настройки статичных файлов

В настройках укажите две переменные.

STATIC_URL - то, по какому адресу будут доступны файлы статики. Его можно делать отличным от того, как называется папка. 
STATICFILES_DIRS - то, в каких названия папок джанга будет искать статичные файлы. Их может быть несколько, но обычно хватает одной "static"

![фвыа](http://images.na4u.ru/static/static/2.png)
### 3. Подключение статических файлов

Обычно стили подключаются в блоке head внтури html в шаблоне. Давайте подключим файл с созданными стилями в books.html

Подключение файла будет происходить с помощью шаблонного тэга sattic. 
Алгоритм такой: 
- вверху шаблона импортировать тэг. 
	- ```
	  {% load static %}
	  ```
- Внутрь пути для подключения файла написать конструкцию вида 
 ```
{% static '/static/main.css' %} 
```

Перейдите на страницу /books. Посмотрите, что цвет шрифта у заголовков изменился на голубой, а значит стили успешно подключились.

Подобным образом подключаются и другие статичные файлы: картинки, js скрипты.
